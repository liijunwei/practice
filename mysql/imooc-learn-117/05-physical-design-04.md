#### 如何选择主键

+ 业务主键: 用于标识业务数据, 进行表与表之间的关联;
+ 数据库主键: 为了优化数据存储(Innodb 会生成6个字节的隐含主键) (???)

+ 根据数据库类型, 考虑主键是否要顺序增长

+ 主键字段的类型所占空间要尽可能小: 对于使用聚集索引方式存储的表, 每个索引后都会附加主键信息(??)
    + 主键小, 每页可加载更多数据, 提升查询效率

#### 避免使用外键约束

+ 降低数据导入的效率

+ 增加维护成本

+ 虽然不建议使用外键约束, 但是相关联的列上一定要建立索引
