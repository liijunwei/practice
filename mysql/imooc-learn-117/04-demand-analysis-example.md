### Example

小型电商网站

#### 核心模块

+ 用户
    + 用途: 记录注册用户的信息
    +
    + 属性: 用户名
    + 属性: 密码
    + 属性: 电话
    + 属性: 邮箱
    + 属性: 身份证号
    + 属性: 地址
    + 属性: 姓名
    + 属性: 昵称
    + 属性: ...
    +
    + 唯一标识: 用户名
    + 唯一标识: 身份证号
    + 唯一标识: 电话
    + 唯一标识: 邮箱
    +
    + 存储特点: 随系统上线时间逐渐增加, 需要永久存储

+ 商品
    + 用途: 记录网站中所销售的商品信息
    +
    + 属性: 商品编码
    + 属性: 商品名称
    + 属性: 商品描述
    + 属性: 商品分类
    + 属性: 供应商名称
    + 属性: 重量
    + 属性: 有效期
    + 属性: 价格
    + 属性: ...
    +
    + 唯一标识: 商品名称+供应商名称, 商品编码
    +
    + 存储特点: 对于下线商品, 可以归档存储
